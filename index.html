Understanding Dijkstra's Algorithm

Dijkstra's Algorithm is a fundamental pathfinding mathematical algorithm conceived by Edsger W. Dijkstra in 1956. It efficiently solves the Single-Source Shortest Path problem, computing the mathematically shortest distance from an initial source node to every single other node in the network.

How It Works Greedily

  1. Initialization: Set the distance to the Source Node to 0, and all other nodes to Infinity. Insert all known nodes into a Priority Queue (a Min-Heap).
  2. Extract Minimum: At each step, extract the node with the absolute smallest known distance. Because no negative edges exist, this guarantees the shortest path to this specific node is securely finalized forever.
  3. Edge Relaxation: Examine all neighbors of the current node. If passing entirely through the current node yields a shorter combined mathematical distance to the neighbor, then "relax" the edge: boldly update the neighboring distance to the improved value, and update its priority in the queue.
  4. Completion: Repeat this systematically until the destination is extracted. Automatically backlight the traversed parent-nodes backwards to perfectly reconstruct your final optimal route tree!

The Negative Weight Limitation

Dijkstra's Algorithm inherently assumes that constantly adding edges to a path can only increase its total length over time. This foundational assumption empowers it to radically freeze a node's shortest distance mathematically once it is extracted from the queue. If negative weights existed, a previously finalized optimal path could suddenly be reduced by a sneaky negative edge later on, violently breaking the algorithm's entire greedy mathematical proof! Dijkstra simply fails when a network generates negative cycles.
